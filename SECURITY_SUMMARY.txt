KEY VULNERABILITIES EXPLAINED
================================================================================

1. EXPOSED API KEY SEVERITY
   The Supabase anon key in Config.swift can be extracted from the compiled
   binary using tools like Frida or by decompiling the app. This gives
   attackers direct access to your backend Edge Functions. If Row-Level
   Security (RLS) is misconfigured, they could:
   - Call Edge Functions without authorization
   - Enumerate/modify place data
   - Perform DOS attacks on your Supabase project

   MITIGATION: Never hardcode secrets. Use Xcode build configurations,
   environment variable injection via CI/CD, or fetch from secure endpoint.

2. UNENCRYPTED LOCATION DATA SEVERITY
   Home location (latitude/longitude) reveals exactly where the user lives.
   Saved places list reveals lifestyle patterns (dispensaries, nightclubs, bars).
   On a jailbroken device or from a backup, this is trivially accessible.

   MITIGATION: Use Keychain with kSecAttrAccessibleWhenUnlockedThisDeviceOnly
   or implement encrypted CoreData.

3. AGE GATE BYPASS SEVERITY
   The app targets alcohol and cannabis products—age verification is a LEGAL
   REQUIREMENT in most jurisdictions. A minor can trivially bypass the client-side
   check by modifying UserDefaults on a jailbroken device.

   MITIGATION: Implement server-side age verification with signed tokens.
   Store verification state in Keychain as interim measure.
